2.1 Input datatables

2.1.1 FBS Consolidated Area Aggregates

Defines the area aggregates to be included in the process.

The composition of the aggregates is defined following the geographicAreaM49 codelist.

Figure 1.1. Datatable presenting the list of area aggregates to be included in the FBS consolidation process.

2.1.2 FBS Consolidated Elements

Defines the input and output elements, along with multipliers for conversion and formulas to compute compound elements.

Used to identify the input elements from the selected output ones, to convert elements to 1000 tonnes, and to compute per-capita elements and the domestic supply quantity.

Figure 1.2. Datatable presenting the list of area aggregates to be included in the FBS consolidation process.

2.1.3 FBS Consolidated Exceptions

Defines the instances to be excluded from the aggregation step and from the dissemination step.

Adapted from a similar table used for dissemination to FAOSTAT: updated codes and labels, and added instances (partial item aggregates and 42 NFISS areas).

Exceptions can be modified upon data ownersâ€™ request.

Figure 1.3. Datatable presenting the list of area aggregates to be included in the FBS consolidation process.

2.1.4 FBS Consolidated Code Mapping

Defines the mapping from input codes to output (consolidated) codes.

The FBS consolidation plugin is robust with respect to possible updates from the old codes to the new codes in the input datasets.

Figure 1.4. Datatable presenting the list of area aggregates to be included in the FBS consolidation process.
